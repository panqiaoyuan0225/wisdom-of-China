# 答题记忆功能实现计划

## 目标
为答题系统添加轻量级记忆功能，记录每道题的答题历史，提升学习效率。

## 功能设计

### 1. 数据存储
新增 `QuestionHistoryManager`，使用 localStorage 存储每道题的：
- `id`: 题目唯一标识（复用 WrongBookManager 的 hash 算法）
- `totalAttempts`: 总答题次数
- `correctCount`: 正确次数
- `wrongCount`: 错误次数
- `lastAttemptTime`: 最后答题时间戳

### 2. UI 展示

#### 2.1 设置页面统计
在设置卡片顶部添加进度统计区域：
- 已完成题目数 / 总题目数
- 总体正确率

#### 2.2 答题时显示历史
在题目编号旁显示小标签：
- 新题：显示「新」
- 已做题目：显示「已做3次 正确率67%」

### 3. 智能出题（可选开关）
新增「优先薄弱题」开关，开启后：
1. 优先推送错题本中的题
2. 其次推送未做过的题
3. 最后推送正确率较低的题

## 实现步骤

### Step 1: 创建 QuestionHistoryManager
- 复用 WrongBookManager 的 generateId 方法
- 实现 getHistory、recordAttempt、getStats 等方法

### Step 2: 在答题流程中记录历史
- 修改 selectOption、submitFill、submitBrief 函数
- 答题后调用 recordAttempt 记录结果

### Step 3: 添加设置页面统计 UI
- 在 settings-card 顶部添加进度统计区域
- 显示已完成/总题目、正确率

### Step 4: 答题时显示题目历史
- 修改 showQuestion 函数
- 在题目编号旁显示历史标签

### Step 5: 添加智能出题功能
- 新增「优先薄弱题」开关
- 修改 startQuiz 函数的出题逻辑
