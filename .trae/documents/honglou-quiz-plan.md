# 红楼梦百题挑战专题计划

## 一、任务概述

在 `honglou_stage3.html` 中添加一个新的专题"拾玖·百题挑战"，仿照 `sanguo_stage1.html` 的百题挑战格式，实现真实的答题交互功能。

## 二、技术方案

### 2.1 导航栏修改
- 在现有6个导航项后添加第7项：`拾玖·百题挑战`
- 编号延续现有格式（拾叁-拾捌之后为拾玖）

### 2.2 题目结构设计

**题型分布：**
| 题型 | 数量 | 题号范围 |
|------|------|----------|
| 选择题 | 50题 | 1-50 |
| 填空题 | 45题 | 51-95 |
| 简答题 | 5题 | 96-100 |

**数据结构：**
```javascript
{ 
    type: '选择/填空/简答', 
    q: '问题内容', 
    options: 'A.xxx | B.xxx | C.xxx | D.xxx', // 仅选择题
    a: '正确答案', 
    note: '解析说明' 
}
```

### 2.3 界面设计

**封面页：**
- 标题：红楼梦 · 百题挑战
- 引言诗句
- "开启挑战"按钮

**答题页：**
- 分区显示：选择题区、填空题区、简答题区
- 选择题：选项按钮，点击即时反馈
- 填空题：输入框 + 提交按钮
- 简答题：文本框 + 完成思考按钮
- 每题配有"查看解析"折叠区

### 2.4 交互功能

| 函数 | 功能 |
|------|------|
| `startQuiz()` | 隐藏封面，显示答题区 |
| `renderQuestions()` | 渲染所有题目 |
| `checkSelect()` | 选择题判题 |
| `checkFill()` | 填空题判题 |
| `checkBrief()` | 简答题处理 |

### 2.5 样式设计

沿用honglou_stage3.html的粉金配色：
- 主色：#b03060（粉红）
- 金色：#d4af37
- 正确反馈：#16a34a（绿色）
- 错误反馈：#ef4444（红色）

## 三、题目内容规划

### 3.1 选择题（50题）覆盖范围
1. 人物关系与身份（10题）
2. 重要情节与事件（15题）
3. 诗词典故（10题）
4. 象征与隐喻（8题）
5. 细节与常识（7题）

### 3.2 填空题（45题）覆盖范围
1. 人物姓名与字号（10题）
2. 地点与建筑（8题）
3. 诗词名句（12题）
4. 重要情节关键词（10题）
5. 象征物与典故（5题）

### 3.3 简答题（5题）覆盖范围
1. 人物形象分析
2. 主题思想探讨
3. 艺术手法赏析
4. 情节意义解读
5. 综合理解题

## 四、实施步骤

1. **添加导航项**：在sidebar中添加"拾玖·百题挑战"
2. **添加CSS样式**：答题区样式、反馈样式、动画效果
3. **添加HTML结构**：封面页和答题区容器
4. **编写题库数据**：100道红楼梦相关题目
5. **实现JavaScript功能**：渲染、判题、反馈逻辑
6. **更新contentData**：将百题挑战内容加入数据对象
7. **测试验证**：确保所有题型正常工作

## 五、预期效果

用户点击"拾玖·百题挑战"后：
1. 首先看到精美的封面页
2. 点击"开启挑战"进入答题
3. 可以真实作答并即时获得反馈
4. 每道题都可查看详细解析
5. 整体风格与红楼梦专题保持一致
